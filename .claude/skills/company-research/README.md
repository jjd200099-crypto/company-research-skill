# 🔬 Company Research Skill for Claude Code

> 把一个投研分析师的完整工作流程，变成一个可复用的 AI 指令模板。

一个 [Claude Code](https://claude.com/claude-code) 自定义 Skill，可以自动化对任何上市公司或 Startup 的深度研究，产出接近机构级质量的研究报告。

## v4 更新亮点

- **一手信息源优先**：上市公司强制先读 2-3 个季度的 Earnings Call Transcript，再搜媒体
- **跨季度分析师追踪**：谁问了什么、管理层怎么答、承诺是否兑现——高密度+可扫描
- **行业专属估值框架**：矿业用 EV/资源量+NAV，银行用 P/TBV+ROTCE，SaaS 用 EV/ARR——不再对所有公司默认 P/E
- **强制同行可比**：每个估值必须包含 3-5 家可比公司对比表
- **事实优先于解读**：分析师问题必须具体到人名+机构+原文引用

## 工作流程

```
Phase 0: 判断公司类型（上市 vs 非上市）
           │                    │
           ▼                    ▼
Phase 1A: 一手信息源       Phase 1B: 多源并行搜索
┌─────────────────────┐   ┌─────────────────────┐
│ ① Earnings Transcript│   │ 英文财务/分析搜索    │
│   （读 2-3 个季度）   │   │ 中文覆盖搜索        │
│ ② 季报/年报新闻稿    │   │ 小红书/Twitter      │
│ ③ Investor Day 材料  │   │ Investor Day 搜索   │
│ ④ 跨季度分析师追踪    │   └─────────────────────┘
└──────────┬──────────┘
           ▼
Phase 2: 深度挖掘（交叉验证媒体 vs 管理层原话）
           ▼
Phase 3: 结构化输出（十章节报告）
```

### 上市公司：Phase 1A 一手信息源优先（强制）

| 步骤 | 内容 | 优先级 |
|------|------|--------|
| Step 1 | 读最近 2-3 个季度的 Earnings Call Transcript **全文** | ★★★★★ |
| Step 2 | 读最新季报/年报新闻稿和 Slide Deck | ★★★★ |
| Step 3 | 读 Investor Day / Capital Markets Day 材料 | ★★★★ |
| Step 4 | 跨季度追踪：分析师反复追问的问题 + 管理层承诺兑现 | ★★★★★ |

**在完成 Phase 1A 之前，不启动任何媒体搜索。**

## 报告结构（十章节）

| 章节 | 核心内容 | 格式要求 |
|------|---------|---------|
| 一、公司概况 | 结构化定位、CEO、业务板块 | 表格为主 |
| 二、最新财务数据 | 多年趋势表、最新季度、指引对比 | 指引需标注「上调/下调/维持」 |
| **三、市场关注焦点与管理层兑现** | **本报告核心章节** | 4-6 主题×密集段落 + 承诺追踪表 + 电话会后变化 |
| 四、核心竞争优势 | 护城河机制+数据+飞轮+竞争对比 | 每个护城河 1-2 段论述 |
| 五、增长驱动力 | 驱动力→受益方式→量化 | 表格 |
| 六、竞争格局 | 可比公司对比表+市占率 | 表格 |
| 七、管理层长期目标 | Investor Day 目标+可信度评估 | 目标表+共识对比表 |
| 八、风险与挑战 | 风险+严重程度+详情 | 表格 |
| **九、估值分析** | **行业专属方法+同行可比+情景分析** | 方法声明+可比表+三情景表 |
| 十、总结 | 现状+拐点+核心风险 | 2-3 句话 |

### 第三节：市场关注焦点与管理层兑现

这是报告中信息密度最高的章节，目标是让读者在 2 分钟内了解：

1. **市场最关注什么** — 4-6 个主题，每个 1 段（~100-150 字）
2. **管理层是否兑现承诺** — 紧凑表格（✅/⏳/❌/🔄）
3. **电话会后发生了什么** — 4-6 条 bullet

每个主题必须包含：具体分析师姓名+机构、管理层原文引用、跨季度演变、当前状态。

❌ 不合格：「分析师关心中国市场」
✅ 合格：「Citi 的 Filippo Falorni 在 Q2 电话会问旅游零售进展，CEO 回答海南 1 月高双位数增长但北京上海机场因零售商过渡有扰动」

### 第九节：行业专属估值框架

| 行业 | 首选方法 | 次选方法 | 避免 |
|------|---------|---------|------|
| 矿业/资源 | EV/资源量, NAV, P/NAV | EV/EBITDA | P/E |
| 银行/金融 | P/TBV, ROTCE, 股息率 | P/E(周期调整) | EV/EBITDA |
| SaaS/软件 | EV/Revenue, Rule of 40 | EV/EBITDA | P/E（亏损期） |
| 消费品/品牌 | EV/EBITDA, P/E(周期调整) | DCF | 单一 P/E |
| 半导体/周期 | EV/EBITDA(mid-cycle), P/BV | 前瞻 P/E | 历史 P/E |
| 油气上游 | EV/EBITDA, P/CF, 股息率 | 油价敏感性 | P/E（油价扭曲） |
| 医药/生物 | rNPV, EV/Pipeline | DCF | P/S |

**每个估值必须：**
1. 明确声明使用的方法和原因
2. 包含 3-5 家可比公司对比表
3. 呈现 Bull / Base / Bear 三情景分析
4. 大宗商品公司附加价格敏感性分析

## 17 条核心规则

1. 每条事实必须附源链接
2. 大量使用表格
3. 结构化事实定位（非口语化类比）
4. **事实优先于解读**——尤其在分析师追踪章节
5. 风险必须诚实披露
6. 估值给明确的框架化答案
7. 默认中文输出
8. 并行搜索最大化效率
9. 追问时做额外搜索而非凭记忆展开
10. **上市公司必须先读一手信息源**
11. **跨季度追踪是强制要求**
12. **Earnings Transcript > 季报 > Investor Day > 分析师报告 > 媒体**
13. Investor Day 目标必须做可信度评估
14. **行业专属估值——不要对所有公司默认 P/E**
15. **三情景估值+商品价格敏感性（资源公司）**
16. 战略投资人信号（客户/合作伙伴入股 = 强验证）
17. **分析师问题必须具体到人名+机构+原文**

## 安装方法

### 方法一：直接下载

```bash
mkdir -p ~/.claude/skills/company-research
curl -o ~/.claude/skills/company-research/skill.md \
  https://raw.githubusercontent.com/jjd200099-crypto/company-research-skill/main/skill.md
```

### 方法二：Git Clone

```bash
cd ~/.claude/skills
git clone https://github.com/jjd200099-crypto/company-research-skill.git company-research
```
